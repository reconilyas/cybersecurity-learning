# Network Ports & Services

---

## 🔌 What is a Port?
A port is a logical endpoint in an operating system used by applications to communicate over a network.

Think of it as a “door” that allows programs to send and receive data.

---

## 📊 Port Ranges

### 🔹 Well-Known Ports (0–1023)
- Used by common system services and protocols

### 🔹 Registered Ports (1024–49151)
- Reserved for specific applications and services

### 🔹 Dynamic / Private Ports (49152–65535)
- Temporary ports used by any application when needed

---

## 🌐 Common Network Ports & Protocols

| Service | Port | Protocol | Description |
|--------|------|----------|-------------|
| FTP    | 20/21 | File Transfer Protocol | File transfers between systems |
| SMTP   | 25   | Simple Mail Transfer Protocol | Sending emails |
| POP3   | 110  | Post Office Protocol | Receiving emails (download-based) |
| IMAP   | 143  | Internet Message Access Protocol | Accessing emails on server |
| NTP    | 123  | Network Time Protocol | Time synchronization |
| NNTP   | 119  | Network News Transfer Protocol | Usenet news |
| RDP    | 3389 | Remote Desktop Protocol | Remote access to systems |
| HTTP   | 80   | HyperText Transfer Protocol | Web traffic |
| HTTPS  | 443  | HTTP Secure (SSL/TLS) | Secure web traffic |

---

## 🧠 Summary
Ports allow different applications to communicate over a network. Each service uses a specific port number to send and receive data efficiently and securely.
